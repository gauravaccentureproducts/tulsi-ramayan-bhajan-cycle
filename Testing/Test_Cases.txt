================================================================================
TULSI-RAMAYAN BHAJAN CYCLE - TEST CASES
================================================================================
Verification suite for every check we run on each kaand before declaring it
done. Use this file as the human-readable authority. Run verify.py in this
same folder for the automatable subset.

Severity legend:
   BLOCKER - must pass; failure means kaand is NOT READY
   MAJOR   - should pass; failure means quality is below the 1950s-studio bar
   MINOR   - nice-to-have; failure means small polish gap

Each test case has:
   ID       - stable identifier
   Scope    - per-chapter / per-kaand / per-cycle
   Severity - BLOCKER / MAJOR / MINOR
   Pass     - what success looks like
   Verify   - how to verify (manual or automated)
   Auto     - YES if verify.py covers this; otherwise manual
   Ref      - section in Procedure_Manual.txt

Usage workflow:
   1. After completing a kaand, run verify.py for automated checks.
   2. Walk through this file for the manual checks.
   3. Fix BLOCKERS first, then MAJORS, then MINORS.

Reference: Tulsi\Procedure_Manual.txt is the source-of-truth manual.
================================================================================

   Severity : BLOCKER

   Verify   : ls / dir of folder; count entries.

   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : Filename matches <KaandName>_Ch<N>_<EpisodeTitleInEnglish>.txt
              N is 1..5; Title uses underscores not spaces.
   Verify   : regex check.
   Auto     : YES
   Ref      : Procedure_Manual §2 Phase 7

TC-S-003 | STYLE/LYRICS section markers present
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : Each chapter file contains exactly one
              '========== STYLE (paste into Suno style box) =========='
              marker AND exactly one
              '========== LYRICS (paste into Suno lyrics box) =========='
              marker, in that order.
   Verify   : substring count check.
   Auto     : YES
   Ref      : Procedure_Manual §2 Phase 7

TC-S-004 | Style prompt under char limit
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : The STYLE block body (between STYLE marker and LYRICS marker)
              is <= 1000 characters including spaces.
   Verify   : char count.
   Auto     : YES
   Ref      : Procedure_Manual §2 Phase 6

TC-S-005 | Verse count per chapter
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : Each chapter has exactly 12 [Verse N] tags, with N from 1 to 12.
   Verify   : grep count of [Verse N] tags.
   Auto     : YES
   Ref      : Procedure_Manual §1.3

TC-S-006 | Chaupai line count
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : Each [Verse N] block has exactly 4 Devanagari lines.
              Total verse lines per chapter = 48 (12 x 4).
              Plus 3 lines of mukhda = 51 lines.
   Verify   : line count between [Verse N] and next tag.
   Auto     : YES

TC-S-007 | Mukhda 3 lines
   Scope    : per-chapter
   Severity : BLOCKER

   Verify   : count lines after opening [Chorus] until blank/next tag.

   Scope    : per-chapter

TC-S-009 | [Outro] tag present

TC-F COMMON RULES (apply to ALL kaands)
------------------------------------------------------------

TC-F-001 | No yuga error
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : The lyrics must NOT contain the word "कलियुग" in any context
              that refers to the Ram-avatar (i.e., as the time when Ravana
              was tyrannizing or when Ram was born). The avatar is in
              Treta Yug.
   Verify   : grep -F "कलियुग" in all chapter lyrics; manual check of context
              if matches found.
   Auto     : YES (keyword presence)
   Ref      : Procedure_Manual §4 anti-pattern #1

TC-F-002 | No cross-avatar conflation (Krishna butter-stealing)
   Scope    : per-chapter
   Severity : BLOCKER
   Pass     : The lyrics must NOT attribute Krishna's signature butter-
              stealing leela to Ram. Banned phrase combinations:
                 मक्खन + लूट / चुरा / चोरी
                 माखन + लूट / चुरा / चोरी
              Ram's bal-leela should use dust-coated brothers, toy
              bow-arrows, anklet-bells, mother's lap imagery.
   Verify   : grep combinations.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #2

TC-F-003 | No Valmiki-style stri-vadh dilemma at Tadaka
   Scope    : Balkand Ch 3 + any future kaand with Tadaka
   Severity : MAJOR
   Pass     : The Tadaka-vadh chaupai should use muni-aagya framing
              ("मुनि-आज्ञा सुनी प्रभु ने" or similar), NOT Valmiki's
              "स्त्री-वध कैसे करें" hesitation framing.
   Verify   : grep "स्त्री-वध" in lyrics body; if present, fail.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #16

TC-F-004 | Ahalya rejoins Gautama
   Scope    : Balkand Ch 4
   Severity : MAJOR
   Pass     : After Ahalya-uddhar, the lyrics must show Ahalya rejoining
              her husband Gautama (e.g., "पति-संग पुनि चली" or "गौतम-स्वामी
              से जा मिलूँगी"), NOT staying alone in solo tap.
   Verify   : grep "पति" near "अहल्या" or "गौतम" in Ch 4.
   Auto     : YES (keyword presence)
   Ref      : Procedure_Manual §4 anti-pattern #15

TC-F-005 | Avatar-katha trio in opening kaand
   Scope    : Balkand Ch 1 (and any kaand opening the cycle with
              avatar-katha)
   Severity : BLOCKER
   Pass     : Lyrics of Ch 1 must contain references to ALL THREE
              Tulsidas-specific avatar-katha episodes:
                 नारद  (Narad-moh)
                 मनु + शतरूपा (Manu-Shatarupa)
                 प्रतापभानु (Pratapbhanu)
   Verify   : grep each name in Ch 1 lyrics.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #6

TC-F-006 | Sumitra-from-both-queens detail
   Scope    : Balkand Ch 2
   Severity : MAJOR
   Pass     : The kheer-vitaran chaupai must explicitly show that Sumitra
              received portions from BOTH Kaushalya AND Kaikeyi. Look for
              "दोनों" + "सुमित्रा" + "भाग" or equivalent.
   Verify   : grep combinations.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #5

TC-F-007 | No forced rhyme padding (banned word list)
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : The following coined / forced-rhyme tokens must NOT appear
              in lyrics:
                 ललाम (filler)
                 हुई हृदय की पाँव (gibberish)
                 शाप-दुख-धारि (coined "धारि")
                 दुष्ट-कमान (Tadaka had no bow)
                 विवाह-गान (coined compound)
                 अनंग used to mean "humble" (अनंग = Kamadeva)
                 कौसल्या सुलक्षणा (reads as proper noun)
   Verify   : grep each token.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-patterns 3, 8, 9, 10, 11

TC-F-008 | All canonical episodes for the kaand present
   Scope    : per-kaand
   Severity : BLOCKER
   Pass     : Every signature episode from this kaand's per-kaand checklist
              (see TC-F-K1 through TC-F-K7 below) is represented by at
              least one chaupai. No invented episodes; no skipped events.
   Verify   : walk-through manually; cross-ref episode keywords vs lyrics.
   Auto     : PARTIAL (keyword spot-check)
   Ref      : Procedure_Manual §3.1

SECTION TC-L - LYRICIST CRAFT
================================================================================

TC-L-001 | AAAA rhyme per chaupai
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : All 4 lines of each chaupai end on the same rhyme syllable.
              No AABB, no ABAB, no ABCB.
   Verify   : extract last word of each line; check vowel-pattern match.
   Auto     : PARTIAL (last-syllable check is approximate; manual review
              for borderline cases recommended).
   Ref      : Procedure_Manual §2 Phase 2

TC-L-002 | Long-vowel rhyme endings
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Each chaupai's rhyme syllable ends in a long vowel
              (-ार, -ान, -ाम, -ास, -ीर, -ंग, -ाल, -ाय, -एत, -ारी, etc.).
              Short-vowel or consonant endings (e.g., -क, -त) are weak.
   Verify   : last-character check per line ending.
   Auto     : YES
   Ref      : Procedure_Manual §3.4

TC-L-003 | No filler/coined words
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : The banned-word list from TC-F-007 is also a lyric-craft
              banned list. No coined-for-rhyme tokens.
   Verify   : keyword grep (same as TC-F-007).
   Auto     : YES
   Ref      : Procedure_Manual §4

TC-L-004 | Stock-word frequency cap
   Scope    : per-chapter
   Severity : MINOR
   Pass     : Across one chapter (51 lines), the following stock words
              should not appear more than the cap:
                 महान       <= 6 times
                 अभिराम     <= 5 times
                 जय-जयकार   <= 4 times
                 धाम        <= 8 times
                 नाम        <= 8 times
                 गान        <= 8 times
   Verify   : word-frequency count.
   Auto     : YES
   Ref      : Procedure_Manual §2 Phase 4

TC-L-005 | Concrete imagery (no abstract padding)
   Scope    : per-chapter
   Severity : MINOR
   Pass     : Lyrics avoid abstract noun-pairs like
                 आशा-धूप, अनुराग छाया, मंगल-मय कल्याण (alone)
              Prefer concrete physical images (lamp, river, ankle-bell,
              dust-coated body, shankh, jaymal).
   Verify   : manual review; flag any line whose meaning is hard to
              picture.
   Auto     : NO
   Ref      : Procedure_Manual §2 Phase 4

TC-L-006 | Register consistency
   Scope    : per-chapter
   Severity : MINOR
   Pass     : Each chaupai sustains one register (devotional-tatsama OR
              rural-folk OR simple-Hindi). Do not mix Sanskrit-heavy
              "विद्या निरुपम भारी" with folk "हँसी की बात" within
              one chaupai.
   Verify   : manual review.
   Auto     : NO
   Ref      : Procedure_Manual §2 Phase 4

TC-L-007 | Mukhda hook power
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : The 3-line mukhda is memorable: clear image OR action
              + Lord's name + jay-ghosh refrain on line 3.
   Verify   : manual; read aloud, ask "would a listener hum this?"
   Auto     : NO
   Ref      : Procedure_Manual §2 Phase 4

TC-L-008 | No grammatical phantom (e.g., सुलक्षणा as name)
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Adjectives must not be misread as proper nouns. If text
              says "तीन रानी कौसल्या सुलक्षणा खास", a listener will
              hear "Sulakshana" as a name. Rewrite.
   Verify   : manual review of any line where an adjective sits between
              two proper nouns or after a name.
   Auto     : NO
   Ref      : Procedure_Manual §4 anti-pattern #3

TC-L-009 | One vivid adjective beats three generic ones
   Scope    : per-chapter
   Severity : MINOR
   Pass     : Avoid stacked generic-adjective patterns like
              "सुन्दर मधुर महान".
   Verify   : grep for 3+ consecutive generic adjectives.
   Auto     : PARTIAL
   Ref      : Procedure_Manual §2 Phase 4

TC-L-010 | Word-sense correctness
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Vocabulary used must match its actual meaning. Specific
              banned misuses:
                 अनंग used for "humble" (अनंग = Kamadeva, body-less love-god)
                 अमीर used in tatsama Hindi context (Urdu word, breaks register)
                 ज़ोर / ज़ (nukta) in devotional verse
   Verify   : keyword grep + context check.
   Auto     : YES (keyword presence)
   Ref      : Procedure_Manual §4 anti-patterns

TC-L-011 | No cliche pairs
   Scope    : per-chapter
   Severity : MINOR
   Pass     : Avoid lazy cliche pairs:
                 "धर्मनिष्ठ हरि-दास"
                 "हो रहा जय-जयकार" (used more than once per chapter)
                 "मन में हर्ष समाए"
   Verify   : grep frequency check.
   Auto     : PARTIAL
   Ref      : Procedure_Manual §4

SECTION TC-M - MUSIC DIRECTOR / SINGABILITY
================================================================================

TC-M-001 | No heavy Sanskrit conjunct cluster at line-end
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Lines must not end on heavy conjunct clusters
              (e.g., -क्त, -ष्ट्र, -क्ष्म). Composer cannot sustain
              the note.
   Verify   : last-syllable conjunct check.
   Auto     : PARTIAL
   Ref      : Procedure_Manual §2 Phase 4

TC-M-002 | No astra-stacking compounds
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Avoid stacked astra names like
                 "ब्रह्मास्त्र-वरुणास्त्र-अग्न्यास्त्र"
              Either one weapon or generic "दिव्य-शस्त्र".
   Verify   : grep for compound astra patterns.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #7

TC-M-003 | No ज़ (nukta) anywhere
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : The character ज़ (ज + nukta) must not appear in lyrics.
              Use देशी ज (without nukta) - more devotional register.
   Verify   : grep for ज़.
   Auto     : YES
   Ref      : Procedure_Manual §4 anti-pattern #9

TC-M-004 | Meter consistency within a chaupai
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : All 4 lines of a chaupai have approximately the same
              matra count (~14-16 matras each). Composer needs a
              regular meter to set bandish.
   Verify   : manual matra-count or syllable count per line.
   Auto     : PARTIAL (syllable-count proxy)
   Ref      : Procedure_Manual §2 Phase 4

TC-M-005 | Open-vowel line endings (sustainability)
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : Lines end on open long vowels (sustainable on Yaman's
              Ga and Ni). Hard consonant stops weaken the bandish.
   Verify   : last-character vowel check.
   Auto     : YES
   Ref      : Procedure_Manual §3.4

TC-M-006 | Tempo and raag stated in style prompt
   Scope    : per-chapter
   Severity : MAJOR
   Pass     : The STYLE block names a specific raag AND a specific tempo
              (vilambit / kaharwa / dadra / medium-brisk / etc.).
   Verify   : keyword presence in STYLE block.
   Auto     : YES
   Ref      : Procedure_Manual §2 Phase 6
